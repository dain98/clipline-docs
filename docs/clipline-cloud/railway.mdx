Railway is the easiest way to run Clipline Cloud. The template deploys the Clipline Cloud app,
Postgres for clip metadata, and an object storage bucket for uploaded media without setting up a VPS
or managing Docker Compose files.

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/clipline-cloud?referralCode=OIcwSe&utm_medium=integration&utm_source=template&utm_campaign=generic)

The latest Clipline Cloud release is
[`v1.2.24`](https://github.com/dain98/clipline-cloud/releases/tag/v1.2.24). The release publishes
`ghcr.io/dain98/clipline-cloud:1.2.24`, `ghcr.io/dain98/clipline-cloud:1.2`, and
`ghcr.io/dain98/clipline-cloud:latest`.

The Railway template can use `latest` for the easy managed path. If you want strict change control,
pin a released image tag in a Docker Compose or VPS deployment.

## Before you start

You need:

- A Railway account.
- A first owner password for the `admin` account.
- The [Clipline desktop app](https://github.com/dain98/clipline) for uploading clips.
- Optional: a custom domain for public clip links and embeds.

The Railway template should create these services:

| Service | Purpose |
|---------|---------|
| Clipline Cloud | Web UI, API, and background processing. |
| Postgres | Durable clip metadata, users, sessions, jobs, and share state. |
| Bucket | S3-compatible object storage for uploaded media. |

## Deploy

1. Click **Deploy on Railway** above.
2. Sign in to Railway and create the project.
3. Enter `CLIPLINE_BOOTSTRAP_ADMIN_PASSWORD`.
4. Start the deployment.
5. Wait for the Clipline Cloud service, Postgres service, and Bucket service to finish deploying.

When deployment finishes, open the Clipline Cloud public domain from Railway. Sign in as:

```text
Username: admin
Password: <the CLIPLINE_BOOTSTRAP_ADMIN_PASSWORD value>
```

Change the owner password after first login, then create normal users from the admin UI.

## Expected wiring

The template should wire the app to Railway Postgres and Railway Object Storage. In the Clipline
Cloud service variables, expect the important values to be present:

```env
CLIPLINE_DATABASE_URL=<Postgres connection URL>
CLIPLINE_STORAGE_BACKEND=s3
CLIPLINE_S3_ENDPOINT=<Bucket endpoint>
CLIPLINE_S3_BUCKET=<Bucket name>
CLIPLINE_S3_REGION=<Bucket region>
CLIPLINE_S3_ACCESS_KEY_ID=<Bucket access key>
CLIPLINE_S3_SECRET_ACCESS_KEY=<Bucket secret key>
CLIPLINE_SESSION_SECRET=<generated secret>
```

Do not upload real clips until `CLIPLINE_DATABASE_URL` and the S3 variables are populated. The bucket
can keep video files after a redeploy, but the library also needs the Postgres records to be durable.
Railway handles the public URL wiring for the service, including custom domains.

## Connect the desktop app

In the Clipline desktop app, add the Railway URL as the Cloud host:

```text
https://<your-railway-domain>
```

Upload a test clip, then confirm:

1. The clip appears in the web library.
2. The clip playback page works.
3. A private link opens for an allowed user.
4. A public share link renders with the right title and preview.

## Custom domain

Railway's generated domain is enough for testing. For a cleaner public install, attach a custom
domain to the Clipline Cloud service in Railway:

1. Open the `dain98/clipline-cloud:latest` service in Railway.
2. Go to **Settings > Networking > Custom Domain**.
3. Add your domain and follow Railway's DNS instructions.
4. Wait for Railway to show the custom domain as active.
5. Open the `dain98/clipline-cloud:latest` service's deployments list.
6. On the currently deployed row, open the three-dot menu on the right.
7. Choose **Redeploy**.

You do not need to set a custom-domain environment variable. Railway provides the public domain
information to the service; redeploying lets Clipline Cloud pick up the updated public address for
desktop client connections, share links, embeds, and callbacks.

## Health checks

Use these routes after deployment:

```text
https://<your-domain>/healthz
https://<your-domain>/readyz
```

`/healthz` means the process is running. `/readyz` means the app can reach both the database and
storage backend.

The admin UI also includes diagnostics for version, storage backend, failed uploads, dead jobs, and
recent audit entries.

## Troubleshooting

| Symptom | Check |
|---------|-------|
| The app starts, but clips disappear after redeploy | Confirm `CLIPLINE_DATABASE_URL` points at Railway Postgres, not SQLite. |
| Uploads fail | Confirm the Bucket variables are present and `CLIPLINE_STORAGE_BACKEND=s3`. |
| Share links use the wrong host | Confirm the custom domain is attached under **Settings > Networking > Custom Domain**, then redeploy the Clipline Cloud service. |
| Desktop app cannot connect | Check `/readyz`, then confirm the desktop app is using the same public HTTPS URL. |
| First login fails | Confirm the password entered during deploy, or redeploy/reset with a known owner password. |

For lower-level Docker Compose and VPS installs, use the
[self-hosting guide](/clipline-cloud/self-hosting).
